- **Provider Behavior**: Document expected behavior for Anthropic, OpenAI,
  Ollama, Custom, and no provider configured.
- **Failure States**: Document player-facing behavior for invalid key, timeout,
  malformed response, provider unavailable, and missing config.
- **Multiplayer Behavior**: Document whether the feature is local-only,
  shared, synchronized, or disabled in multiplayer.
- **Spoiler Behavior**: Document whether spoilers are possible and how default
  spoiler avoidance is preserved.
- **In-Game Acceptance**: Include at least one acceptance scenario that uses the
  feature through normal Stardew Valley interaction.
